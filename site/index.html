Evidence middleware

The missing middleware between CAE simulations, LLM agents, and the research record.

CaeReflex turns solver folders, mesh files, VTK outputs, scripts, notes, and DOI metadata into structured, provenance-aware evidence that agents can inspect and cite without pretending to validate the physics.

  • ▧Unify engineering context. Normalize simulation assets, scripts, notes, and publication metadata into one inspectable case record.
  • ▧Preserve provenance. Track which files and metadata sources support each claim so humans can audit the evidence trail.
  • ▧Constrain agent claims. Help LLM workflows cite context clearly while warning that CaeReflex does not validate solver physics.

01

Problem, implication, solution.

The homepage now mirrors the README's core argument: fragmented CAE evidence creates unsafe partial context for agents, so CaeReflex normalizes the record first.

Problem

Evidence is fragmented

Gmsh geometry, OpenFOAM dictionaries, VTK outputs, scripts, notes, and DOI searches can all describe one engineering question while living in incompatible places.

Implication

Agents see partial context

An LLM assistant may receive a hand-written summary instead of the files, warnings, provenance, and research metadata needed for a careful answer.

Solution

Normalize before reasoning

CaeReflex inspects in read-only mode and exports ReflexCase JSON, compact agent context, reports, flags, and optional CrossRef metadata.

02

A stronger evidence-flow model.

Raw CAE artefacts become a portable engineering record before they are handed to notebooks, reports, APIs, or LLM tools.

Compact context, Markdown reports, BibTeX, and REST/OpenAPI actions.

Review

Human + LLM workflow

Agents summarize inspected evidence while humans verify physics and decisions.

Optional CrossRef rail: DOI metadata and available abstracts can be attached only when explicitly requested, adding research context without implying full-paper reading or engineering proof.
03

Trust boundary, shown early.

CaeReflex earns confidence by saying exactly what it can and cannot do before an agent produces conclusions.

CaeReflex is

  • a read-only inspection and documentation layer;
  • a provenance-aware case record generator;
  • a bridge from CAE evidence to LLM-readable context;
  • a safer tool layer for local agent workflows.

CaeReflex is not

  • a solver or simulation runner;
  • a validation or certification system;
  • a convergence, mesh-quality, or physical-correctness proof;
  • a replacement for qualified engineering judgement.

05

Why it is different.

Compared with ad-hoc upload, generic parsing, or ungrounded RAG, CaeReflex preserves engineering provenance and safety labels.

Read-only CAE inspection

Adapters inspect supported artefacts without solver execution or file mutation.

Provenance-aware records

ReflexCase separates source files, detected facts, inferred context, and warnings.

Agent-native outputs

Export JSON, compact context, reports, BibTeX, and REST/OpenAPI actions.

CrossRef grounding

Attach DOI metadata and available abstracts only when the user explicitly asks.

Safety by design

Inspection flags and do-not-claim rules make overconfident agent summaries easier to catch.

Single-file site

The landing page remains framework-free, fast, and suitable for static hosting.

07

Choose a quickstart path.

No hosted backend is required. Start locally from the CLI, expose REST/OpenAPI, or connect an agent workflow.

CLI inspection

Inspect and export

Install, import a bundled case, and write JSON, agent context, and a Markdown report.

pip install -e ".[server]"
caereflex inspect examples/openfoam_cavity_minimal \
  --out caereflex.json \
  --agent-context agent_context.json \
  --report case_report.md

REST/OpenAPI

Serve local actions

Run the local server and give tool-calling agents a discoverable OpenAPI document.

caereflex serve \
  --host 127.0.0.1 \
  --port 8765
open http://127.0.0.1:8765/openapi.yaml

09

Clear safety boundary.

This is a trust feature. CaeReflex helps inspection and documentation, but it does not replace engineering validation.

Do not claim that CaeReflex:

  • runs solvers;
  • validates simulations;
  • proves convergence;
  • assesses mesh adequacy;
  • certifies engineering results;
  • approves design safety;
  • reads full papers from CrossRef metadata;
  • replaces qualified engineering judgement.
