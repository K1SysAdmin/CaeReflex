# Learning Paths

Use these paths to match the curriculum to the learner's role. The projects are modular, so teams can run a short workshop or a deeper onboarding sequence.

## Path A: CAE engineer

Goal: use CaeReflex to inspect local artefacts and prepare evidence for review.

1. [CLI-first Inspection](cli-first-inspection.md)
2. [OpenFOAM Case Review](openfoam-case-review.md)
3. [VTK Result Context](vtk-result-context.md)
4. [CrossRef Literature Context](crossref-literature-context.md)
5. [REST/OpenAPI Agent Workflow](rest-openapi-agent-workflow.md), optional

## Path B: Python or platform developer

Goal: understand the software surfaces and how workflows move through CLI, services, adapters, exporters, and REST.

1. [CLI-first Inspection](cli-first-inspection.md)
2. [REST/OpenAPI Agent Workflow](rest-openapi-agent-workflow.md)
3. [Adapter Extension Design](adapter-extension-design.md)
4. [Assessment Rubric](assessment-rubric.md), for review standards

## Path C: AI-agent engineer

Goal: expose engineering evidence to an agent safely and avoid unsafe tool claims.

1. [CLI-first Inspection](cli-first-inspection.md)
2. [REST/OpenAPI Agent Workflow](rest-openapi-agent-workflow.md)
3. [CrossRef Literature Context](crossref-literature-context.md)
4. [Glossary](glossary.md), especially agent context, provenance, extracted fact, and inferred fact

## Path D: Instructor or technical lead

Goal: run the curriculum for a team or class.

1. [Instructor Guide](instructor-guide.md)
2. [Environment Setup](environment-setup.md)
3. [CLI-first Inspection](cli-first-inspection.md)
4. Pick two domain projects: Gmsh, OpenFOAM, VTK, or CrossRef
5. Finish with [Assessment Rubric](assessment-rubric.md)

## Path E: Advanced contributor

Goal: extend CaeReflex without breaking its architecture or safety boundary.

1. [Adapter Extension Design](adapter-extension-design.md)
2. [Architecture: Adapters](../architecture/adapters.md)
3. [Architecture: Services Layer](../architecture/services-layer.md)
4. [Developer Guide: Adding Adapters](../developer-guide/adding-adapters.md)
5. [Developer Guide: Testing](../developer-guide/testing.md)
